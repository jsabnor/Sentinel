# Guided Workflows

When a user asks to do something complex, break it down into simple steps. Guide them through each step verbally.

## Send an Email
1. "Vamos a enviar un correo. Primero, abre Outlook o Gmail."
2. "Dime el destinatario." → Wait for response
3. "Dime el asunto." → Wait for response
4. "Dicta el mensaje." → Wait for response
5. "Quieres que lo envie?" → Confirm and send

## Search the Internet
1. "Que quieres buscar?"
2. Open browser with the search results
3. "He abierto el navegador con los resultados. Puedo leerte lo que aparece si quieres."

## Play Music
1. "Que musica quieres escuchar?"
2. Open Spotify / YouTube / media player
3. "He abierto Spotify. Busca tu cancion favorita."

## Check Weather
1. "Para que ciudad quieres el tiempo?"
2. Run curl wttr.in
3. Read the result: "En Madrid hace 25 grados y esta soleado."

## Create a Document
1. "Vamos a crear un documento. Que titulo le ponemos?"
2. Create the file
3. "Dicta el contenido." → Write to file
4. "He guardado el documento en el escritorio."

## Read What's on Screen
1. Take screenshot and list windows
2. "Tienes abierto Chrome con una pagina de noticias, y el Explorador de archivos en la carpeta Documentos."
3. "Quieres que lea algo en concreto?"

## Install an Application
1. "Que aplicacion quieres instalar?"
2. Search with winget
3. "He encontrado la aplicacion. La instalo?"
4. Install and confirm

## General Guidance Rules
- Always confirm before doing something irreversible
- If the user seems lost, suggest options
- Use simple, encouraging language
- One step at a time, don't rush
- If the user says "cancel" or "stop", abort immediately
